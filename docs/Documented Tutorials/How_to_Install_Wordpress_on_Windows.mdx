## Overview

This comprehensive guide covers the complete installation and configuration of WordPress on Windows using a modern, professional stack: PHP 8.x, MariaDB, and Caddy web server. This setup provides excellent performance, security, and maintainability for both development and production environments.

### What You'll Build

- **WordPress CMS**: Latest version with professional configuration
- **PHP 8.x**: Modern PHP runtime with optimized settings
- **MariaDB**: High-performance MySQL-compatible database
- **Caddy Web Server**: Automatic HTTPS and reverse proxy
- **NSSM Service Manager**: Professional Windows service management

### Architecture Overview

```
Internet → Caddy (Port 80/443) → PHP-FPM (Port 9000) → WordPress → MariaDB (Port 3306)
```

### Key Benefits

- **Professional Setup**: Enterprise-ready configuration
- **Automatic HTTPS**: SSL certificates managed by Caddy
- **High Performance**: Optimized PHP and database settings
- **Security Hardened**: Best practices throughout
- **Service Management**: Automatic startup and monitoring
- **Scalable**: Ready for production workloads

## Prerequisites

Before beginning the installation, ensure your system meets all requirements:

### System Requirements

#### Minimum Requirements
- **Operating System**: Windows 10 64-bit (version 1903) or Windows 11
- **RAM**: 4GB system memory (8GB+ recommended)
- **Storage**: 10GB available disk space (SSD recommended)
- **Processor**: 64-bit processor (2+ cores recommended)

#### CLI Download Requirements
- **Windows 10 version 1803+** or **Windows 11** for built-in `curl` and `tar` commands
- **For older Windows versions**: Use the manual download alternatives provided in each section

#### Recommended Requirements
- **Operating System**: Windows 11 or Windows 10 Pro (latest version)
- **RAM**: 16GB+ system memory
- **Storage**: SSD with 50GB+ available space
- **Processor**: Multi-core 64-bit processor (4+ cores)
- **Network**: Stable internet connection

### Software Prerequisites

- **Administrator Access**: Required for service installation
- **PowerShell 5.1+**: For automation scripts
- **Visual C++ Redistributable**: For PHP and MariaDB
- **Domain Name**: For production deployment (optional for development)

### Network Requirements

- **Ports 80 and 443**: Available for web traffic
- **Port 3306**: Available for database (can be changed)
- **Port 9000**: Available for PHP-FPM
- **Firewall**: Configured to allow necessary traffic

### Step 2: Install and Configure PHP

#### Extract and Setup PHP

Extract PHP

```cmd
REM Extract PHP using built-in tar command
echo Extracting PHP...
%SystemRoot%\system32\tar.exe -xf "%TEMP%\php-latest-nts-Win32-vs16-x64.zip" -C "C:\Tools"
echo Moving PHP files to correct location...
for /d %%i in ("C:\Tools\php-*-nts-Win32-vs16-x64") do (
    xcopy "%%i\*" "C:\Tools\PHP\" /E /I /Y /Q
    rmdir "%%i" /S /Q
)
echo PHP extracted to C:\Tools\PHP
```

Copy php.ini template
```cmd
REM Copy php.ini template
copy "C:\Tools\PHP\php.ini-production" "C:\Tools\PHP\php.ini"
echo PHP configuration template copied
```

### Step 3: Install and Configure MariaDB

#### Install MariaDB

Install MariaDB silently

```cmd
REM Install MariaDB silently
echo Installing MariaDB...
msiexec /i "%TEMP%\mariadb-latest-winx64.msi" /quiet SERVICENAME=MariaDB PASSWORD=SecureRootPassword123! UTF8=1
echo ✅ MariaDB installed

REM Wait for service to start
timeout /t 10 /nobreak
```

#### Configure MariaDB

Create WordPress database and user

```cmd
REM Create database setup script
echo CREATE DATABASE wordpress CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; > "%TEMP%\setup_db.sql"
echo CREATE USER 'wordpress'@'localhost' IDENTIFIED BY 'WordPressPassword123!'; >> "%TEMP%\setup_db.sql"
echo GRANT ALL PRIVILEGES ON wordpress.* TO 'wordpress'@'localhost'; >> "%TEMP%\setup_db.sql"
echo FLUSH PRIVILEGES; >> "%TEMP%\setup_db.sql"

REM Execute database setup (find MariaDB installation dynamically)
for /d %%i in ("C:\Program Files\MariaDB*") do (
    "%%i\bin\mysql.exe" -u root -p"SecureRootPassword123!" < "%TEMP%\setup_db.sql"
    echo ✅ WordPress database created
)

REM Clean up
del "%TEMP%\setup_db.sql"
```

#### Secure MariaDB Installation

Create secure configuration

```cmd
REM Get MariaDB installation path dynamically and create configuration
for /d %%i in ("C:\Program Files\MariaDB*") do (
    REM Create MariaDB configuration file
    echo [mysqld] > "%%i\data\my.ini"
    echo # Basic Settings >> "%%i\data\my.ini"
    echo bind-address = 127.0.0.1 >> "%%i\data\my.ini"
    echo port = 3306 >> "%%i\data\my.ini"
    echo datadir = %%i/data >> "%%i\data\my.ini"
    echo. >> "%%i\data\my.ini"
    echo # Performance Settings >> "%%i\data\my.ini"
    echo innodb_buffer_pool_size = 1G >> "%%i\data\my.ini"
    echo innodb_log_file_size = 256M >> "%%i\data\my.ini"
    echo innodb_flush_log_at_trx_commit = 2 >> "%%i\data\my.ini"
    echo innodb_flush_method = O_DIRECT >> "%%i\data\my.ini"
    echo. >> "%%i\data\my.ini"
    echo # Security Settings >> "%%i\data\my.ini"
    echo skip-name-resolve >> "%%i\data\my.ini"
    echo sql_mode = STRICT_TRANS_TABLES,NO_ZERO_DATE,NO_ZERO_IN_DATE,ERROR_FOR_DIVISION_BY_ZERO >> "%%i\data\my.ini"
    echo. >> "%%i\data\my.ini"
    echo # Character Set >> "%%i\data\my.ini"
    echo character-set-server = utf8mb4 >> "%%i\data\my.ini"
    echo collation-server = utf8mb4_unicode_ci >> "%%i\data\my.ini"
    echo. >> "%%i\data\my.ini"
    echo # Logging >> "%%i\data\my.ini"
    echo log-error = %%i/data/error.log >> "%%i\data\my.ini"
    echo slow_query_log = 1 >> "%%i\data\my.ini"
    echo slow_query_log_file = %%i/data/slow.log >> "%%i\data\my.ini"
    echo long_query_time = 2 >> "%%i\data\my.ini"
    echo ✅ MariaDB configuration updated at %%i\data\my.ini
)
```

### Step 4: Setup NSSM Service Manager

#### Extract and Configure NSSM
Extract NSSM

```cmd
REM Extract NSSM using built-in tar command
echo Extracting NSSM...
%SystemRoot%\system32\tar.exe -xf "%TEMP%\nssm-latest.zip" -C "%TEMP%"
```

Copy appropriate version

```cmd
REM Find and copy NSSM executable (works with any version)
for /d %%i in ("%TEMP%\nssm-*") do (
    if exist "%%i\win64\nssm.exe" (
        copy "%%i\win64\nssm.exe" "C:\Tools\NSSM\nssm.exe"
        echo ✅ NSSM copied to C:\Tools\NSSM\
    )
)
```

Add NSSM to PATH
```cmd
REM Add NSSM to system PATH (check for duplicates first)
echo %PATH% | find /i "C:\Tools\NSSM" >nul
if errorlevel 1 (
    setx PATH "%PATH%;C:\Tools\NSSM" /M
    echo ✅ NSSM added to system PATH
) else (
    echo ✅ NSSM already in system PATH
)
```

#### Create PHP-FPM Service

```cmd
REM Install PHP as a service using NSSM
echo Creating PHP-FPM service...
"C:\Tools\NSSM\nssm.exe" install PHP-FPM "C:\Tools\PHP\php-cgi.exe"
"C:\Tools\NSSM\nssm.exe" set PHP-FPM Parameters "-b 127.0.0.1:9000"
"C:\Tools\NSSM\nssm.exe" set PHP-FPM AppDirectory "C:\Tools\PHP"
"C:\Tools\NSSM\nssm.exe" set PHP-FPM DisplayName "PHP FastCGI Process Manager"
"C:\Tools\NSSM\nssm.exe" set PHP-FPM Description "PHP FastCGI Process Manager for WordPress"

REM Configure service recovery
"C:\Tools\NSSM\nssm.exe" set PHP-FPM AppExit Default Restart
"C:\Tools\NSSM\nssm.exe" set PHP-FPM AppRestartDelay 5000

REM Start PHP-FPM service
"C:\Tools\NSSM\nssm.exe" start PHP-FPM
echo ✅ PHP-FPM service created and started
```
