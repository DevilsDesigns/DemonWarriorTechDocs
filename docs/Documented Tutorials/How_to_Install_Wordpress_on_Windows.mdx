## Overview

This comprehensive guide covers the complete installation and configuration of WordPress on Windows using a modern, professional stack: PHP 8.x, MariaDB, and Caddy web server. This setup provides excellent performance, security, and maintainability for both development and production environments.

### What You'll Build

- **WordPress CMS**: Latest version with professional configuration
- **PHP 8.x**: Modern PHP runtime with optimized settings
- **MariaDB**: High-performance MySQL-compatible database
- **Caddy Web Server**: Automatic HTTPS and reverse proxy
- **NSSM Service Manager**: Professional Windows service management

### Architecture Overview

```
Internet → Caddy (Port 80/443) → PHP-FPM (Port 9000) → WordPress → MariaDB (Port 3306)
```

### Key Benefits

- **Professional Setup**: Enterprise-ready configuration
- **Automatic HTTPS**: SSL certificates managed by Caddy
- **High Performance**: Optimized PHP and database settings
- **Security Hardened**: Best practices throughout
- **Service Management**: Automatic startup and monitoring
- **Scalable**: Ready for production workloads

## Prerequisites

Before beginning the installation, ensure your system meets all requirements:

### System Requirements

#### Minimum Requirements
- **Operating System**: Windows 10 64-bit (version 1903) or Windows 11
- **RAM**: 4GB system memory (8GB+ recommended)
- **Storage**: 10GB available disk space (SSD recommended)
- **Processor**: 64-bit processor (2+ cores recommended)

#### Recommended Requirements
- **Operating System**: Windows 11 or Windows 10 Pro (latest version)
- **RAM**: 16GB+ system memory
- **Storage**: SSD with 50GB+ available space
- **Processor**: Multi-core 64-bit processor (4+ cores)
- **Network**: Stable internet connection

### Software Prerequisites

- **Administrator Access**: Required for service installation
- **PowerShell 5.1+**: For automation scripts
- **Visual C++ Redistributable**: For PHP and MariaDB
- **Domain Name**: For production deployment (optional for development)

### Network Requirements

- **Ports 80 and 443**: Available for web traffic
- **Port 3306**: Available for database (can be changed)
- **Port 9000**: Available for PHP-FPM
- **Firewall**: Configured to allow necessary traffic

## Installation Process

### Step 1: Create Directory Structure

Create main tools directory

```powershell
$toolsPath = "C:\Tools"
New-Item -ItemType Directory -Path $toolsPath -Force
```

Create subdirectories
```powershell
$directories = @(
    "$toolsPath\PHP",
    "$toolsPath\NSSM", 
    "$toolsPath\Caddy",
    "$toolsPath\WordPress",
    "C:\inetpub\wwwroot\wordpress"
)

foreach ($dir in $directories) {
    New-Item -ItemType Directory -Path $dir -Force
    Write-Host "Created: $dir"
}
```

### Step 2: Install and Configure PHP

#### Extract and Setup PHP

Extract PHP

```powershell
# Find your downloaded PHP file (works with any PHP 8.x version)
$phpZip = Get-ChildItem "C:\Temp\php-8.*-nts-Win32-vs16-x64.zip" | Sort-Object Name -Descending | Select-Object -First 1 -ExpandProperty FullName
if (-not $phpZip) {
    Write-Error "PHP zip file not found in C:\Temp. Please check the file location."
    exit 1
}
Write-Host "Extracting PHP from: $(Split-Path $phpZip -Leaf)"
Expand-Archive -Path $phpZip -DestinationPath "C:\Tools\PHP" -Force
```

Copy php.ini template
```
Copy-Item "C:\Tools\PHP\php.ini-production" "C:\Tools\PHP\php.ini"
```

#### Configure PHP Settings

Create optimized php.ini configuration
```powershell
$phpIniPath = "C:\Tools\PHP\php.ini"
$phpConfig = @"
; PHP Configuration for WordPress
; Memory and Performance
memory_limit = 512M
max_execution_time = 300
max_input_time = 300
post_max_size = 64M
upload_max_filesize = 64M
max_file_uploads = 20

; Error Reporting (adjust for production)
display_errors = Off
log_errors = On
error_log = C:\Tools\PHP\logs\php_errors.log

; Extensions
extension_dir = "ext"
extension=bz2
extension=curl
extension=fileinfo
extension=gd
extension=gettext
extension=intl
extension=mbstring
extension=exif
extension=mysqli
extension=openssl
extension=pdo_mysql
extension=zip
extension=opcache

; OPcache Configuration
opcache.enable=1
opcache.enable_cli=1
opcache.memory_consumption=256
opcache.interned_strings_buffer=8
opcache.max_accelerated_files=10000
opcache.revalidate_freq=2
opcache.fast_shutdown=1

; Session Configuration
session.save_handler = files
session.save_path = "C:\Tools\PHP\tmp"
session.gc_maxlifetime = 1440

; CGI Configuration
cgi.force_redirect = 0
cgi.fix_pathinfo = 1
fastcgi.impersonate = 1
"@

# Apply configuration
$phpConfig | Out-File -FilePath $phpIniPath -Encoding UTF8 -Append
```

#### Create PHP Directories

Create required directories

```powershell
$phpDirs = @(
    "C:\Tools\PHP\logs",
    "C:\Tools\PHP\tmp",
    "C:\Tools\PHP\sessions"
)

foreach ($dir in $phpDirs) {
    New-Item -ItemType Directory -Path $dir -Force
}
```

#### Add PHP to System PATH

Add PHP to system PATH

```powershell
$currentPath = [Environment]::GetEnvironmentVariable("PATH", [EnvironmentVariableTarget]::Machine)
if ($currentPath -notlike "*C:\Tools\PHP*") {
    $newPath = $currentPath + ";C:\Tools\PHP"
    [Environment]::SetEnvironmentVariable("PATH", $newPath, [EnvironmentVariableTarget]::Machine)
    Write-Host "✅ PHP added to system PATH"
}

# Verify PHP installation
& "C:\Tools\PHP\php.exe" --version
```

### Step 3: Install and Configure MariaDB

#### Install MariaDB

Install MariaDB silently

```powershell
# Find your downloaded MariaDB installer (works with any version)
$mariaInstaller = Get-ChildItem "C:\Temp\mariadb-*-winx64.msi" | Sort-Object Name -Descending | Select-Object -First 1 -ExpandProperty FullName
if (-not $mariaInstaller) {
    Write-Error "MariaDB installer not found in C:\Temp. Please check the file location."
    exit 1
}
Write-Host "Installing MariaDB from: $(Split-Path $mariaInstaller -Leaf)"

$installArgs = @(
    "/i", $mariaInstaller,
    "/quiet",
    "SERVICENAME=MariaDB",
    "PASSWORD=SecureRootPassword123!",
    "UTF8=1"
)

Start-Process -FilePath "msiexec.exe" -ArgumentList $installArgs -Wait -NoNewWindow
Write-Host "✅ MariaDB installed"
```

#### Configure MariaDB

Wait for MariaDB service to start

```powershell
Start-Sleep -Seconds 10
```

Create WordPress database and user

```powershell
$mysqlCommands = @"
CREATE DATABASE wordpress CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'wordpress'@'localhost' IDENTIFIED BY 'WordPressPassword123!';
GRANT ALL PRIVILEGES ON wordpress.* TO 'wordpress'@'localhost';
FLUSH PRIVILEGES;
"@
```

```
Execute MySQL commands (Dynamic path detection)
$mariaDBPath = Get-ChildItem "C:\Program Files\MariaDB*" | Sort-Object Name -Descending | Select-Object -First 1 -ExpandProperty FullName
$mysqlExe = "$mariaDBPath\bin\mysql.exe"
$mysqlCommands | & $mysqlExe -u root -p"SecureRootPassword123!"
Write-Host "✅ WordPress database created"
```

#### Secure MariaDB Installation

Create secure configuration

```powershell
# Get MariaDB installation path dynamically
$mariaDBPath = Get-ChildItem "C:\Program Files\MariaDB*" | Sort-Object Name -Descending | Select-Object -First 1 -ExpandProperty FullName
$mariaDataPath = "$mariaDBPath\data"

$mariaConfig = @"
[mysqld]
# Basic Settings
bind-address = 127.0.0.1
port = 3306
datadir = $($mariaDataPath -replace '\\', '/')
```

```powershell
# Performance Settings
innodb_buffer_pool_size = 1G
innodb_log_file_size = 256M
innodb_flush_log_at_trx_commit = 2
innodb_flush_method = O_DIRECT
```

Security Settings
```powershell
skip-name-resolve
sql_mode = STRICT_TRANS_TABLES,NO_ZERO_DATE,NO_ZERO_IN_DATE,ERROR_FOR_DIVISION_BY_ZERO
```
Character Set

```
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci
```

Logging

```
log-error = $($mariaDataPath -replace '\\', '/')/error.log
slow_query_log = 1
slow_query_log_file = $($mariaDataPath -replace '\\', '/')/slow.log
long_query_time = 2
"@

$mariaConfigPath = "$mariaDataPath\my.ini"
$mariaConfig | Out-File -FilePath $mariaConfigPath -Encoding UTF8
Write-Host "✅ MariaDB configuration updated at: $mariaConfigPath"
```

### Step 4: Setup NSSM Service Manager

#### Extract and Configure NSSM
Extract NSSM

```powershell
# Find your downloaded NSSM file (works with any version)
$nssmZip = Get-ChildItem "C:\Temp\nssm-*.zip" | Sort-Object Name -Descending | Select-Object -First 1 -ExpandProperty FullName
if (-not $nssmZip) {
    Write-Error "NSSM zip file not found in C:\Temp. Please check the file location."
    exit 1
}
Write-Host "Extracting NSSM from: $(Split-Path $nssmZip -Leaf)"
Expand-Archive -Path $nssmZip -DestinationPath "C:\Temp\nssm-temp" -Force
```
Copy appropriate version

```powershell
# Find NSSM executable (works with any version)
$nssmSource = Get-ChildItem "C:\Temp\nssm-temp\nssm-*\win64\nssm.exe" | Select-Object -First 1 -ExpandProperty FullName
if (-not $nssmSource) {
    Write-Error "NSSM executable not found in extracted files."
    exit 1
}
Copy-Item $nssmSource "C:\Tools\NSSM\nssm.exe"
Write-Host "✅ NSSM copied to C:\Tools\NSSM\"
```

Add NSSM to PATH
```
$currentPath = [Environment]::GetEnvironmentVariable("PATH", [EnvironmentVariableTarget]::Machine)
if ($currentPath -notlike "*C:\Tools\NSSM*") {
    $newPath = $currentPath + ";C:\Tools\NSSM"
    [Environment]::SetEnvironmentVariable("PATH", $newPath, [EnvironmentVariableTarget]::Machine)
    Write-Host "✅ NSSM added to system PATH"
}
```

#### Create PHP-FPM Service

```powershell
# Install PHP as a service
& "C:\Tools\NSSM\nssm.exe" install PHP-FPM "C:\Tools\PHP\php-cgi.exe"
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM Parameters "-b 127.0.0.1:9000"
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM AppDirectory "C:\Tools\PHP"
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM DisplayName "PHP FastCGI Process Manager"
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM Description "PHP FastCGI Process Manager for WordPress"

# Configure service recovery
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM AppExit Default Restart
& "C:\Tools\NSSM\nssm.exe" set PHP-FPM AppRestartDelay 5000

# Start PHP-FPM service
& "C:\Tools\NSSM\nssm.exe" start PHP-FPM
Write-Host "✅ PHP-FPM service created and started"
```
